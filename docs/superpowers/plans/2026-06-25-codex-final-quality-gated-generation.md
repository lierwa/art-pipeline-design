# Codex Final Quality-Gated Generation Implementation Plan

> **Superseded:** This guide-driven whole-redraw plan is kept only as a postmortem record.

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rework Codex final generation into a hatch-pet-aligned candidate pipeline where Codex only creates raw candidates, deterministic Python code handles measurement/postprocess/QA, and bad candidates never overwrite accepted final assets.

**Architecture:** Backend owns job preparation, role-labelled input artifacts, raw candidate provenance, deterministic transparent postprocess, quality gate, and final promotion. Codex worker owns only AI visual generation and returns an original generated image path plus a short QA note. Layout guides are generated by Python from cleaned analysis masks and source geometry, then passed to Codex as optional layout-only references.

**Tech Stack:** FastAPI backend, Pydantic manifest models, Pillow-only deterministic image analysis, existing WorkspaceTask/Codex final task APIs, existing frontend workbench review UI.

---

## Background

The failed iteration mixed three different responsibilities:

- Codex visual generation.
- Local deterministic postprocess.
- Layout correction / quality recovery.

That made bad outputs look like model failures even when the local pipeline caused part of the damage. The observed failures fall into separate buckets:

- **Squeezed or distorted final:** local postprocess must never non-uniformly resize or force a candidate into an incompatible canvas.
- **Blurred final:** local processing must avoid repeated resampling and must flag low-effective-resolution candidates before promotion.
- **Wrong grouping / wrong angle / wrong relative layout:** Codex must receive source/mask/guide as role-labelled grounding inputs, and QA must block obvious layout drift.
- **Mask residue affecting guide math:** guide measurement must use a cleaned analysis mask, not the raw user-visible mask.

This plan borrows the hatch-pet contract, not the pet domain:

- AI generation returns raw visual candidates.
- Parent/backend records provenance and owns manifests.
- Python scripts/modules do deterministic extraction, validation, and packaging/promotion only.
- Layout guides are construction references, not identity sources and not final outputs.
- Deterministic validation is necessary but not sufficient; visual/AI review remains part of the repair loop.

## Target Flow

1. **Prepare job**
   - Freeze `source_crop`, `transparent_cutout`, `mask`, `generation_brief`, prompt hint, previous final if present, and failed candidate if this is a repair.
   - Create a job folder like `elements/element_031/codex_final/job/job_20260625051631371468_59e6da64/`.

2. **Build analysis mask**
   - Read raw `mask` or `transparent_cutout` alpha.
   - Create an internal cleaned mask for measurement only.
   - Remove tiny connected components, ignore isolated far-away specks, and keep the dominant subject region.
   - Do not overwrite the user-visible SAM2/manual mask.

3. **Generate layout guide**
   - Python creates `layout_guide.png`.
   - The guide may include canvas bounds, safe area, center lines, cleaned-mask bbox, cleaned-mask centroid, and coarse isometric axis hints.
   - The guide must not infer component labels like A/B/C or assume a fixed object count.
   - The guide is passed to Codex only as `layout_guide`.

4. **Generate raw candidate**
   - Codex worker receives the prompt and all role-labelled images.
   - Codex generates one RGB image on chroma background.
   - Worker returns a concrete generated source path and a one-sentence QA note, for example `selected_source=/Users/guojunxi/.codex/generated_images/019efd3a-9847-7a73-9404-5280ea9e5f37/ig_20260625_051640.png` and `qa_note=The candidate preserves the source crop angle and has a clean chroma background.`

5. **Record raw candidate**
   - Backend validates the selected source is an original generated image, then copies it to `codex_raw.png`.
   - Raw candidate is preserved even if QA later fails.

6. **Deterministic postprocess**
   - Remove chroma background.
   - Crop transparent border.
   - Fit with aspect ratio preserved.
   - Save job-local candidate output.
   - Do not write the canonical `transparent_asset.png` yet.

7. **Quality gate**
   - Check alpha is non-empty.
   - Check subject is not clipped against output edges.
   - Check visible area and bbox are not extreme outliers against source/mask.
   - Check effective candidate resolution is not too low.
   - Check final aspect ratio was not distorted.
   - Check chroma residue / background alpha pollution.
   - Check near-copy of SAM2 cutout.

8. **Promote or block**
   - If QA passes, copy job-local candidate output to canonical `codex_final/transparent_asset.png`, update state, task, manifest, and metadata.
   - If QA fails, mark the job as failed or review-needed, keep the previous final asset untouched, and write a short repair note.

9. **Repair**
   - Regenerate only failed assets.
   - Repair input includes source crop, mask/cutout, previous accepted final, failed candidate, layout guide, and repair note.
   - User prompt hint remains a short failure description, not the hard constraint system.

## Non-Goals

- Do not add a second generation pass to turn a candidate into final.
- Do not ask users to write full prompt contracts.
- Do not use layout guide to fix blur, squeezing, or bad postprocess.
- Do not infer grouped asset component names or counts from labels like A/B/C.
- Do not trust noisy raw mask bbox as the final layout truth.
- Do not add OpenCV or other new image-processing dependencies for the first version.

- [ ] **Step 2: Implement Pillow-only analysis mask**

Implement:

```python
@dataclass(frozen=True)
class CodexFinalAnalysisMask:
    bbox: tuple[int, int, int, int] | None
    centroid: tuple[float, float] | None
    visible_area: int
    kept_component_count: int
    removed_component_count: int
    image: Image.Image
```

Algorithm:

- convert alpha or grayscale to binary
- flood-fill connected components with 8-neighbor connectivity
- sort by area
- keep components above an area threshold
- always keep the largest component
- remove components whose area is tiny and whose centroid is far from the largest component
- compute bbox/centroid from kept pixels

Add Chinese WHY comment explaining that this mask is for measurement only and must not mutate user-visible masks.

## Acceptance Criteria

- Layout guide is deterministically generated by Python, not AI.
- Analysis mask cleaning happens before guide generation and never overwrites user-visible masks.
- Codex final worker receives source/mask/cutout/brief/guide as role-labelled inputs.
- Raw selected source is preserved.
- Final PNG is not produced by a second AI pass.
- Postprocess never non-uniformly stretches candidates.
- Candidate output is job-local until QA passes.
- Failed candidate does not overwrite the previous accepted final.
- QA report explains why a candidate was blocked.
- Repair reruns only failed assets and uses source, previous final, failed candidate, and repair note as grounding.

## Postmortem

This plan was superseded on 2026-06-26 by `2026-06-26-codex-final-fidelity-first-cleanup.md`.
Browser and manual review showed that guide-driven whole redraw made grouped assets unstable:
small assets drifted inside oversized canvases, grouped objects changed relative layout, and
parent assets could be globally repainted or simplified. The layout guide, failed-candidate
feedback, and previous-final feedback are therefore removed from the normal generation path.

The replacement contract is fidelity-first: normal assets use deterministic source-crop +
accepted-mask candidates, and AI is used only for bounded parent repair regions. The UI now
labels these checks as blocking checks rather than visual QA acceptance.
